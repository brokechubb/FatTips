# Contributing to FatTips

Thank you for your interest in contributing to FatTips! This document provides guidelines for contributing.

## Development Setup

1. Fork the repository
2. Clone your fork
3. Follow the [Quick Start](../README.md#quick-start) guide
4. Create a branch for your changes

## Code Style

- TypeScript for all code
- Follow existing code patterns
- Use meaningful variable names
- Add comments for complex logic

## Commit Messages

Use conventional commits:
- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation
- `test:` - Tests
- `refactor:` - Code refactoring

Example: `feat: add support for multiple tokens`

## Pull Request Process

1. Ensure tests pass
2. Update documentation
3. Reference related issues
4. Request review from maintainers

## Reporting Issues

- Use issue templates
- Provide reproduction steps
- Include error messages
- Specify environment (OS, Node version, etc.)

## Questions?

Join our [Discord](https://discord.gg/yourserver) or open a discussion.
